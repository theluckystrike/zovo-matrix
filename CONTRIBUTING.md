# Contributing to zovo-matrix

Thanks for your interest in contributing to the Matrix. Here's how you can help.

## Submitting Issues

- Use the GitHub issue tracker to report bugs or suggest features.
- Include steps to reproduce the problem, your macOS version, and terminal app.
- Screenshots are welcome — drop them in your issue.

## Submitting Pull Requests

1. Fork the repo and create your branch from `main`.
2. Make your changes and test them locally.
3. Write a clear PR description explaining what you changed and why.
4. Submit the PR and wait for review.

## Code Style

- Keep it clean, keep it green, keep it Matrix.
- Comment your code where intent isn't obvious.
- Follow existing patterns and naming conventions in the project.

## Be Kind

This is an open-source project built for fun and privacy. Be respectful, be helpful, and remember — we're all in the Matrix together.
